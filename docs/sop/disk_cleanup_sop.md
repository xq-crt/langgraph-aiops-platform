# 磁盘空间清理 SOP（只读扫描 + 人工确认）

## 1. 确认告急分区
- 查看各挂载点使用率与剩余 GB
- 区分「空间满」与「inode 满」

## 2. 定位大目录
- 对用户目录 / 数据盘做有限深度目录聚合 TopN
- 排除系统卷影、回收站等敏感路径的深入扫描

## 3. 定位大文件
- 按阈值（如 100MB）列出 Top 大文件
- 核对是否为日志、缓存、安装包、开发产物

## 4. 清理建议（禁止自动删除）
- Temp、Downloads、回收站、Docker 日志、开发缓存等需人工确认
- 生产环境走变更窗口；禁止对正在写入的日志直接 `rm`（应 truncate 或轮转）

## 5. 验证
- 清理后复查分区使用率
- 观察 24h 增长趋势，排除泄漏型增长
